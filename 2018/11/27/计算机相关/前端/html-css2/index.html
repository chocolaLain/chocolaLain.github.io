HTML框架

  1. 框架标签（frame）：
    • 框架对于页面的设计有着很大的作用
  2. 框架集标签（<frameset>）：
    • 框架集标签定义如何将窗口分割为框架
    • 每一个frameset定义一系列行或列
    • rows/cols的值规定了每行或每列占据屏幕的面积

      frame和frameset稍微落后了一些
      使用frameset的时候要去掉body标签,而且使用frameset和frame标签的时候会出现删除或者下划线,表示他们已经过时(弹幕上说是全都阵亡在PHP了)

  3. 常用标签：
    • noresize：固定框架大小
    • cols：列
    • rows：行
  4. 内联框架(重要):iframe
    • iframe可以再body里面
    • frameborder可以去掉外边框
    • 也可以设置高度和宽度
    • 如果某个框架内有很多内容,会显示进度条,可上下滑动浏览
    • 框架的嵌套和与超链接的使用
    • 超链接target可以设置_blank(新页面打开),_self(自身打开), _parent(父框架打开), _top(顶部框架打开)

HTML背景

  • 背景标签:background
  • 背景颜色:bgcolor
  • 颜色:
    • 颜色是由一个十六进制符号来定义，这个符号由红色、绿色和蓝色的值组成（RGB）
    • 颜色值最小值：O（#00）
    • 颜色值最大值：255（#FF)
    • 红色：#FF0000
    • 绿色：#00FF00
    • 蓝色：#0000FF

XHTML介绍

简介:

  • 什么是XHTML:
    • XHTML指的是可扩展超文本标记语言
    • XHTML与HTML4.01几乎是相同的
    • XHTML是更严格更纯净的HTML版本
    • XHTML是以XML应用的方式定义的HTML
    • XHTML得到所有主流浏览器的支持
  • 为什么使用XHTML?
    • 为了代码的完整性和良好性
  • 文档声明:
    • DTD(文档类型定义(Document Type Definition)):规定了使用通用标记语言的网页语法
    • xsd，dtd，tld有什么区别和联系？
    • STRICT（严格类型）
      TRANSITIONAL（过渡类型）
      FRAMESET（框架类型）
  • 还涉及到了XHTML的一些声明,但只做了解
  • 加了XHTML声明,一些不规范的代码就会被提示,例如元素不完整,或者标签格式不规范.
  • XHTML元素语法:
    • XHTML元素必须正确嵌套:例如h1 p content /h1 /p,这是个错误的嵌套
      XHTML元素必须始终关闭:例如h1 /h1 或者 br/,要有’/‘来关闭
      XHTML元素必须小写:例如不允许”A”标签的写法
      XHTML文档必须有一个根元素:例如body header这种根元素
  • XHTML属性语法规则:
    • XHTML属性必须使用小写:例如不允许’WIDTH’这种属性写法
      XHTML属性值必须用引号包围:例如width=”100px”
      XHTML属性最小化也是禁止的:不允许简写,例如type=button,不能直接去写button

HTML5与HTML4区别

  • 推出的理由及目标:
    HTML5的出现，对于Web来说意义是非常重大的，因为他的意图是想要把目前Web上存在的各种问题一并解决掉了。
    • Web浏览器之间的兼容性很低
    • 文档结构不够明确
    • Web应用程序的功能受到了限制

世界知名浏览器厂商对HTML5的支持:微软、Google、苹果、Opera、Mozilla

  • 内容的改变:

    • 内容类型:文件扩展名,html或者htm没有变
    • DOCTYPE声明:HMTL4中需要明确指出是哪个版本,5中直接!DOCTYPE(可小写) HTML
    • 指定字符编码:

    • 可以省略标记的元素

    • 具有boolean值的属性:例如checkbox这样的属性,不指定属性值的时候默认为true,如果想要为false,可以不写该属性.

    • 省略引号:指定属性值的时候可以去掉引号.

  • 新增的元素和废除的元素

    新增的结构元素:
    点击显/隐内容
    • section: 页面中的一个内容块,如章节 页眉 页脚以及页面中的其他部分,可与h1~h6标题结合使用,表示文档的结构
    • article: 表示页面中的一块与上下文不相关的独立内容,例如博客或者报纸当中的一篇文章
    • aside: 表示article元素之外的,与article相关的辅助信息
    • header: 页面中的一个内容区域块,通常表示标题
    • hgroup: 用于对整个页面或者一个内容块的标题进行集合
    • footer:表示页面中的一个部分,一个区域块,通常来表示这个区域块的底部 脚部,可以用来承载作者 姓名日期等与作者相关的元素
    • nav: 用于页面中导航链接的一部分
    • figure:表示一段独立的流的内容
    新增的其他元素:
    • video:表示电影片段或者视频流
    • audio:定义音乐,例如音频或者音频流
    • cavans(画布):表示图形,例如图表或者其他图形,这个元素本身没有行为,只是提供一个画布,但是可以借助API来画图
      新增的input元素的类型
    • email:表示email地址
    • url:表示文本框输入的一个地址
    • number:表示输入的数字
    • range:表示输入框内输入的一个范围值
    • Date Pickers:表示日历的日期 时间
      废除的元素:
    • 能使用CSS替代的元素：basefont,big,center,font,s,tt,u等
    • 不再使用frame框架
    • 只有部分浏览器支持的元素
    • 其他被废除的元素

新增的属性和废除的属性
新增的属性:

  • 表单相关的属性
  • 链接相关的属性
  • 其他属性

全局属性

  • contentEditable属性
    可以规定内容是否可以编辑,是布尔值类型
  • designMode属性
    也是规定页面是否可以编辑,但是只能在js文档里面设置,on可编辑,off不可编辑
  • hidden属性
    可以用于隐藏元素
  • spellcheck属性
    可以进行拼写检查,例如contentable设置好后,在设置这个,文本框内输入appple,会提示你拼写错误
  • tabindex属性
    可以利用table键将焦点放在设置了tabindex的元素上,例如给链接设置了,这样可以方便打开链接.
    tabindex=”1”,可以给每个元素设置数字选择先选中哪个元素
    如果不需要某个元素来tab,但将来可能会用到,可以设置值为”-1”,这样tab时不会选中,但是程序中是可以获取到tabindex的.

新增的结构元素

  • article元素
    • article元素代表文档、页面或应用程序中独立的、完整的、可以独自被外部引用的内容。它可以是一篇博客或者报刊中的文章，一篇论坛帖子、一段用户评论或独立的插件，或其他任何独立的内容。
    • article元素是可以嵌套使用的。例如一篇博客文章中的评论就可以使用嵌套article元素的方式
    • article元素可以用来表示插件
    • 代替了DIV之后语义化更加方便,不过从内容的呈现角度上来说是没有区别的
      点击显/隐内容
      还涉及到了object标签和embed标签
  • section元素
    • section元素用于对网站或应用程序中页面上的内容进行分块。一个section元素通常由内容及其标题组成。但section元素并非一个普通的容器元素；当一个容器需要被直接定义样式或通过脚本定义行为时，推荐使用div而非section元素。
    • 不推荐给没有标题和内容的元素使用section,主要用于给页面上的内容分块或者给文章分段,不要与有着完整的独立的内容的article元素混淆.
    • 注意的是section可以和article嵌套使用,但前者侧重于分段分块,后者侧重于独立,section也要有个标题,当使用一些样式的时候,偏向于使用div
